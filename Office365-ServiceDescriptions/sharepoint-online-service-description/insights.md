# <a name="insights"></a>Аналитика

SharePoint Online это набор средств и технологий, основанных на веб-технологиях, которые помогают организациям хранить, совместно использовать цифровые данные и управлять ими. Эта размещенная служба, основанная на Microsoft SharePoint Server 2013, идеально подходит для работы с проектами, хранения данных и документов в централизованном расположении и обмена информацией с другими. Следующие возможности Insights позволяют организациям использовать информацию в базах данных, отчетах и бизнес-приложениях для решения определенных бизнес-задач.
  
## <a name="business-intelligence-center"></a>Центр бизнес-аналитики

Сайт центра бизнес-аналитики позволяет хранить подключения к данным, книги, отчеты, системы показателей и панели мониторинга в централизованном, безопасном месте. Узнайте больше о [настройке центра бизнес-аналитики](https://go.microsoft.com/fwlink/p/?LinkId=270883).
  
## <a name="calculated-measures-and-members"></a>Вычисляемые показатели и элементы

При создании отчетов сводной диаграммы из отчетов сводной таблицы, которые используют данные, хранимые в службах SQL Server Analysis Services, вы можете создавать вычисляемые показатели и элементы для отчета. Дополнительные сведения см. в разделе [Возможности бизнес-аналитики в службах Excel](https://go.microsoft.com/fwlink/p/?LinkId=271025).
  
## <a name="data-connection-library"></a>Библиотека подключений к данным

Библиотека подключений к данным — это специальная библиотека SharePoint, созданная для хранения общих подключений к данным и управления ими. К ним относятся ODC-файлы, созданные с помощью Excel. Узнайте больше о [надежных библиотеках подключений к данным в SharePoint Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=271026).
  
## <a name="decoupled-pivottables-and-pivotcharts"></a>Разъединенные сводные таблицы и диаграммы

При создании отчета сводной диаграммы автоматически создается соответствующий отчет сводной таблицы. В Excel 2013 Нет необходимости отображать отчет сводной таблицы на том же листе, что и соответствующий отчет сводной диаграммы. Вы можете отображать только отчет сводной диаграммы. Эта возможность называется *несвязанными сводными таблицами и сводными диаграммами*. Узнайте больше о [сводных таблицах и сводных диаграммах](https://go.microsoft.com/fwlink/?LinkId=271027).
  
## <a name="excel-services"></a>Службы Excel

Службы Excel позволяют предоставлять доступ к книгам другим пользователям, взаимодействовать с отчетами, просматривать и просматривать данные в окне браузера и обновлять данные. Службы Excel также позволяют предоставлять общий доступ ко всем или частям книг, а также управлять тем, какие сведения отображаются в этих книгах. В SharePoint 2016 возможности служб Excel доступны через Excel для Интернета на сервере Office Online Server. Этот сервер должен быть установлен в локальной среде. Узнайте больше о [службах Excel](https://go.microsoft.com/fwlink/p/?LinkId=271028).
  
## <a name="excel-web-access-web-part-for-sharepoint"></a>Веб-часть Excel Web Access для SharePoint

С помощью веб-части Excel Web Access можно отображать содержимое Excel на сайте SharePoint
  
## <a name="field-list-and-field-support"></a>Поддержка списка полей и полей

Excel для Интернета позволяет открыть список полей и поля для отчетов сводных диаграмм и отчетов сводных таблиц, которые просматриваются в окнах браузера. Эта возможность позволяет временно изменить данные, которые отображаются в отчете сводной диаграммы или таблицы, не открывая Excel. Узнайте больше о [бизнес-аналитике в службах Excel](https://go.microsoft.com/fwlink/p/?LinkId=271029).
  
## <a name="filter-enhancements"></a>Улучшенные фильтры

Теперь стало проще создавать срезы и подключать их к отчетам. Вы можете настроить размер и ориентацию среза. Вы также можете форматировать срезы, применяя встроенные стили или определив собственный стиль.
  
## <a name="filter-search"></a>Фильтрация поиска

Вы можете выполнять поиск в результатах фильтра, что упрощает переход к определенному элементу в длинном списке. Дополнительные сведения см. в разделе [Использование фильтров и срезов](https://go.microsoft.com/fwlink/p/?LinkId=271030).
  
## <a name="odc-file-support"></a>Поддержка ODC-файлов

ODC-файлы позволяют управлять подключениями к корпоративным источниками данных, совместно использовать их и повторно использовать. Подключения к данным с определенными источниками данных могут настраиваться централизованно и управляться ИТ-персоналом и предоставляться для любой книги, использующей ODC-файлы. Вы можете хранить наборы ODC-файлов в Центральных расположениях в ферме SharePoint, а также изменять книги Excel для использования этих файлов.
  
## <a name="performancepoint-services"></a>PerformancePoint Services

Не доступно для клиентов SharePoint Online. PerformancePoint Services позволяет клиентам SharePoint Server 2013 создавать интерактивные панели мониторинга, отображающие ключевые индикаторы производительности (KPI) и виртуализацию данных в виде систем показателей, отчетов и фильтров. Узнайте больше о [службах PerformancePoint Services](https://go.microsoft.com/fwlink/p/?LinkId=271032).
  
## <a name="power-bi-web-part"></a>Веб-часть Power BI

Power BI это облачный набор инструментов бизнес-аналитики, позволяющий всем подключаться к данным, визуализировать и анализировать данные с большим быстродействием, эффективностью и пониманием. На удобных информационных панелях, в интерактивных отчетах и привлекательных визуализациях данные оживают. Чтобы узнать больше о Power BI, ознакомьтесь [со статьей начало работы с Power BI Service](https://go.microsoft.com/fwlink/?linkid=844671).
  
## <a name="power-pivot-for-excel-in-sharepoint"></a>Power Pivot для Excel в SharePoint

С помощью Power Pivot для Excel можно выполнять мощный анализ данных и создавать сложные модели данных. Кроме того, вы можете собирать большие объемы данных из различных источников, быстро их анализировать и делиться идеями. В SharePoint Online можно использовать Excel для Интернета, чтобы просматривать диаграммы, таблицы, отчеты сводных диаграмм и отчеты сводных таблиц в окне браузера и взаимодействовать с ними. Кроме того, вы можете использовать веб-часть Excel Web Access, чтобы отобразить контент Excel на сайте SharePoint. Прежде чем просматривать листы Power Pivot в SharePoint, необходимо создать их в Excel. Узнайте больше об использовании [Power Pivot в Excel](https://go.microsoft.com/fwlink/p/?LinkId=271036).
  
## <a name="power-view-for-excel-in-sharepoint"></a>Power View для Excel в SharePoint

 Надстройка Power View для Excel позволяет визуализировать смоделированные данные и взаимодействовать с ними с помощью интерактивных визуализаций, анимаций и интеллектуальных запросов. В SharePoint Online для просмотра и взаимодействия с книгами, содержащими листы Power View в окне браузера, используется Excel для веб-сайта. Кроме того, вы можете использовать веб-часть Excel Web Access, чтобы отобразить контент Excel на сайте SharePoint. Прежде чем просматривать листы Power View в SharePoint, необходимо создать их в Excel. Узнайте больше об использовании [Power View в Excel](https://go.microsoft.com/fwlink/p/?LinkId=271035).
  
## <a name="quick-explore"></a>Экспресс-тенденции

Краткий обзор позволяет организациям выбрать значение в отчете сводной диаграммы или отчете сводной таблицы и просмотреть дополнительные сведения об этом значении. Например, если вы просматриваете отчет о продажах и выбираете значение, представляющее общую сумму продаж, с помощью экспресс-тенденций можно увидеть дополнительную информацию, например сумма продаж по разным категориям продуктов или географическим областям. Узнайте больше о [кратком обзоре](https://go.microsoft.com/fwlink/p/?LinkId=271037).
  
## <a name="refresh-excel-workbooks-interactively"></a>Интерактивное обновление книг Excel

Позволяет просматривать актуальные данные в Excel для веб-книг, подключенных к внешним источникам данных (например, базам данных MS SQL). Обратите внимание, что книги необходимо подключить к внешним источникам данных в Excel, прежде чем просматривать их в Excel для Интернета.
  
## <a name="sql-server-reporting-services-ssrs-integrated-mode"></a>Интегрированный режим служб SQL Server Reporting Services (SSRS)

Если службы SQL Server 2012 Reporting Services интегрированы с SharePoint, вы можете использовать широкие возможности совместной работы и централизованного управления документами SharePoint для контента служб SQL Server 2012 Reporting Services. Отчеты хранятся в библиотеках документов SharePoint наряду с другими отчетами, например Excel для веб-файлов. С помощью разрешений SharePoint можно управлять доступом к контенту служб Reporting Services, а пользователи смогут запускать конструктор отчетов, конструктор моделей и построитель отчетов в библиотеке документов. Узнайте больше о [сервере отчетов служб Reporting Services](https://go.microsoft.com/fwlink/p/?LinkId=271039).
  
## <a name="sql-server-reporting-services-ssrs-web-part"></a>Веб-часть SQL Server Reporting Services (SSRS)

Веб-часть служб SQL Server 2012 Reporting Services позволяет пользователям отображать существующий отчет Reporting Services в собственной веб-части на сайте SharePoint. Если в отчете Reporting Services содержатся параметры, пользователи могут подключаться к этому отчету с помощью параметров веб-части. Дополнительные сведения о [веб-части "средство просмотра отчетов](https://go.microsoft.com/fwlink/p/?LinkId=271041)".
  
## <a name="timeline-slicer"></a>Срез временной шкалы

Срез временной шкалы — это элемент управления, который можно добавить в панель мониторинга с помощью Excel. Элемент управления Timeline позволяет пользователям просматривать сведения в течение определенного периода времени. Узнайте больше о [бизнес-аналитике в службах Excel](https://go.microsoft.com/fwlink/p/?LinkId=271042).
  
## <a name="use-and-interact-with-excel-workbooks-containing-data-models"></a>Работа с книгами Excel, содержащими модели данных

Некоторые книги Excel содержат модели данных Power Pivot, позволяющие подробно и быстро анализировать данные, совмещать большие объемы данных из различных источников и делиться результатами анализа.
  
## <a name="user-defined-functions"></a>Пользовательские функции

Пользовательские функции (UDF) предоставляют возможность использовать формулы в Excel для ячеек веб-листа для вызова пользовательских функций, написанных в управляемом коде. UDF — это расширенная функция программирования, позволяющая корпоративным организациям разрабатывать и подключать пользовательский код в Excel для веб-листов. Вы можете создавать пользовательские функции, чтобы вызывать пользовательские математические функции, получать данные из пользовательских источников данных на листы и вызывать веб-службы из пользовательских функций.
  
## <a name="visio-services"></a>Службы Visio

Службы Visio — это приложение-служба, которое позволяет пользователям совместно использовать и просматривать файлы\*документа Microsoft Visio (vsdx) и веб-\*документа Visio 2010 (VDW-файлы). Службы Visio больше не доступны в SharePoint Online. Службы Visio по прежнему будут доступны в SharePoint Server 2010, SharePoint Server 2013 и SharePoint Server 2016. Узнайте больше о [службах Visio](https://go.microsoft.com/fwlink/p/?LinkID=271043).

Чтобы просмотреть доступность функций в планах Office 365, отдельных и локальных решениях, ознакомьтесь с [описанием службы SharePoint Online](sharepoint-online-service-description.md).
